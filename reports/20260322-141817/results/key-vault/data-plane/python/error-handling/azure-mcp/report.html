## Key Points:

**1. Inspecting HttpResponseError:**
```python
except HttpResponseError as e:
    status = e.status_code        # The HTTP status code (403, 404, 429)
    message = e.message            # Human-readable error message
    error_code = e.error.code      # Azure error code
    error_details = e.error.message # Detailed error info
```

**2. Status Code Handling:**
- **403 (Access Denied)**: Missing RBAC role - grant "Key Vault Secrets User" role
- **404 (Not Found)**: Secret doesn't exist OR is soft-deleted
- **429 (Throttling)**: Check `Retry-After` header and implement exponential backoff

**3. Soft-Deleted Secrets:**
When you call `get_secret()` on a soft-deleted secret:
- You get a **404 error** (it appears "not found")
- To access it, use `get_deleted_secret(name)` to inspect
- Use `begin_recover_deleted_secret(name).result()` to restore it
- After recovery, `get_secret()` works normally

**4. Best Practices:**
- Use `ResourceNotFoundError` (subclass of HttpResponseError) to specifically catch 404s
- Implement retry logic with exponential backoff for 429 errors
- Check `response.headers.get('Retry-After')` for throttling guidance
- For batch operations, handle errors per-secret to continue processing
