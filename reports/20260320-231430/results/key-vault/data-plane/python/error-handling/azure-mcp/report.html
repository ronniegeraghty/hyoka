## Key Points:

**1. Exception Types:**
- `ResourceNotFoundError` - 404 errors (secret not found)
- `HttpResponseError` - All HTTP errors with `status_code` property
- `ServiceRequestError` - Network/connection issues

**2. Inspecting Errors:**
```python
except HttpResponseError as e:
    e.status_code          # HTTP status (403, 404, 429, etc.)
    e.message              # Error description
    e.error.code           # Azure error code
    e.response.headers     # Response headers (e.g., Retry-After)
```

**3. Common Scenarios:**
- **403 (Access Denied)**: Missing RBAC role like "Key Vault Secrets User"
- **404 (Not Found)**: Secret doesn't exist OR is soft-deleted
- **429 (Throttling)**: Check `Retry-After` header for wait time

**4. Soft-Deleted Secrets:**
When soft-delete is enabled:
- `get_secret()` raises `ResourceNotFoundError` (404)
- Use `get_deleted_secret()` to access soft-deleted secrets
- Retrieve `deleted_date` and `scheduled_purge_date`
- Can recover with `begin_recover_deleted_secret()`

**5. Best Practices:**
- Implement exponential backoff for 429 errors
- Don't retry on 404/403 (permanent failures)
- Use the `Retry-After` header when present

The example includes retry logic, existence checking, and detailed error inspection patterns you can adapt for your application!
